# Problem statement

The user wants to change how archive ignore rules are maintained for runtime-generated archive artifacts.

Today, the archive directory `.gitignore` entry for `.specd-index.jsonl` is created during project initialization. The user considers this the wrong lifecycle point because the archive runtime can later create or recreate the index file independently of init.

The desired outcome is:

- ensure the archive directory `.gitignore` is updated when `.specd-index.jsonl` itself is created or maintained
- ensure `.staging` under the archive directory is also ignored
- make ignore maintenance resilient when the archive directory is moved, recreated, or otherwise not preserved from the original init step

# Approach / solution outline

Proposed direction discussed:

- move archive-specific `.gitignore` maintenance away from `project init` as the primary mechanism
- perform or guarantee archive `.gitignore` maintenance from archive runtime operations associated with `.specd-index.jsonl`
- add both of these entries to the archive directory `.gitignore`:
  - `.specd-index.jsonl`
  - `.staging`

The working assumption is that the runtime behavior belongs closer to `FsArchiveRepository` and/or the archive storage adapter rather than only `FsConfigWriter.initProject`.
