# update-archive-reporting ‚Äî Exploration Context

## Problem Statement

The `ArchiveChange` contract was recently updated to include more metadata, but the CLI commands `archive show` and `archive list` do not yet take full advantage of this. `archive list` specifically has a redundant `WORKSPACE` column and lacks pagination, which is necessary as the archive grows. We need to improve metadata display and implement efficient listing with pagination.

## Approach / Solution Outline

- **`archive show`**: Update to display `specIds` (already in contract) and improve formatting.
- **`archive list`**:
  - Remove the legacy `WORKSPACE` field.
  - Implement pagination with `--limit` (default 100), `--page`, and `--start-at` (exclusive).
  - Display a summary line: "Showing X archived changes out of Y. Increase limit or specify another page."
- **Performance**: Introduce `.specd-index-meta.json` next to `.specd-index.jsonl` to store the total count of archived changes, avoiding full index scans for metadata.
- **`core` support**: Update `ListArchived` use case and `ArchiveRepository` port to support pagination options.

## Affected Areas

- **Packages**: `cli`, `core`
- **Commands**: `archive show`, `archive list`
- **Use Cases**: `ListArchived`, `ArchiveChange` (to update meta on archive)
- **Entities/Models**: `ArchivedChangeIndexEntry`
- **Infrastructure**: `FsArchiveRepository` (index management, meta file)

## Spec IDs

- `cli:archive-show`: Update output display.
- `cli:archive-list`: Remove `WORKSPACE`, add pagination requirements and summary line.
- `core:archive-change`: Update to maintain `.specd-index-meta.json`.
- `core:list-archived`: Update to support pagination input.
- `core:archive-repository-port`: Add pagination to `list()` and meta management.

## Design Decisions

- Remove `WORKSPACE` column.
- Use `.specd-index-meta.json` for total count.
- Pagination flags: `--limit`, `--page`, `--start-at`.
- Default limit: 100.

## Agreements Reached

- Avoid adding `description` to the index for now as it's not currently prioritized and would increase index size without a strong use case yet.
- Focus on performance and usability for large archives.

## Key Codebase Observations

- `ArchiveRepository` already handles the `.specd-index.jsonl`.
- `index.jsonl` lines are parsed as `IndexEntry`.

## User Preferences

- Clean, paginated output.
- Performance first.
