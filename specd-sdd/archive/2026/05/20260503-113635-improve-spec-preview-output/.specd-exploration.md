# Exploration Context: improve-spec-preview-output

## Problem Statement

The current output of `specd changes spec-preview` is ambiguous when an artifact is incomplete, a no-op, or the spec does not belong to the change. Users need clearer feedback about why a preview might look like the original or why it's missing.

## Approach / Solution Outline

- **Core (`PreviewSpec` use case):**
  - Modify `PreviewSpecFileEntry` to include a `status` field: `merged`, `no-op`, or `missing`.
  - `no-op`: Delta exists but results in no changes (all entries are `no-op`).
  - `missing`: Artifact file is expected by schema but not found in change artifacts or delta application failed.
  - `PreviewSpec` should return entries for all expected artifacts of a spec, even if missing or no-op, to allow the CLI to report them properly.
- **CLI (`spec-preview` command):**
  - Update text output to handle the new statuses:
    - `missing` (delta): `--- <filename> --- (missing artifact, showing original)` and show original content.
    - `missing` (new spec): `--- <filename> --- (missing artifact)` and show nothing.
    - `no-op` (delta): `--- <filename> --- (no-op delta, showing original)` and show original content.
  - Improve error message for `SpecNotInChangeError` to suggest `specd specs show`.

## Design Decisions

- `PreviewSpec` will now include entries for "missing" or "no-op" artifacts instead of silently skipping them.
- CLI will use these statuses to provide the requested labels in the header.

## Agreements Reached

- If a delta is `missing`, show the original content with the label `(missing artifact, showing original)`.
- If a delta is `no-op`, show the original content with the label `(no-op delta, showing original)`.
- If a new spec is `missing`, show only the header with `(missing artifact)`.
- If the spec is not in the change, suggest `specd specs show`.

## Key Codebase Observations

- `PreviewSpec` currently skips no-op deltas and missing files.
- `SpecNotInChangeError` message is generic and lacks the suggestion.
- `spec-preview.ts` in CLI handles the header formatting.

## User Preferences

- Specific labels requested: `(missing artifact, showing original)`, `(missing artifact)`, `(no-op delta, showing original)`.
- Explicit suggestion for `specd specs show` on spec-not-in-change error.
