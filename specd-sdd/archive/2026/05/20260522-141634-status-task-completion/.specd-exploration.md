# Exploration: status-task-completion

Generated: 2026-05-22

## Problem statement

Artifacts with `hasTasks = true` (like `tasks`) show `[hasTasks]` in the CLI DAG view but don't display how many tasks are completed vs total. The user wants to see `[hasTasks - 10/30 done]` or similar inline.

## Solution outline

- Use `taskCompletionCheck.incompletePattern` and `completePattern` (already defined in schema `ArtifactType`) to count task items from artifact file content
- Add a new optional field `taskCompletion: { complete, incomplete, total }` to `ArtifactStatusEntry` in `GetStatus`
- In `GetStatus.execute()`, for each schema artifact with `hasTasks`, read artifact files via `ChangeRepository.artifact()` and count matches against the patterns
- In CLI `status.ts` (renderDag), change `[hasTasks]` to `[hasTasks - N/M done]` using the status entry data
- Also show task counts in the details section

- `cli:src/commands/change/status.ts` — `renderDag()` function, details section

## Design decisions

- Reuse `safeRegex` from `../../domain/services/safe-regex.js` (same as `TransitionChange._checkTaskCompletionForArtifact`)
- Counting logic mirrors `TransitionChange._checkTaskCompletionForArtifact` — match `completePattern` for complete, `incompletePattern` for incomplete, total = complete + incomplete
- `taskCompletion` is optional — only present when schema artifact type has `hasTasks: true` and `taskCompletionCheck` is configured
- If artifact files don't exist or fail to load, skip silently (no completion data)

## Key codebase observations

- `ArtifactType.hasTasks()` and `ArtifactType.taskCompletionCheck()` provide schema-level config
- `TaskCompletionCheck` has `incompletePattern?: string` and `completePattern?: string` (validation-rule.ts:31-34)
- `ChangeRepository.artifact(change, filename)` loads file content
- `safeRegex(pattern, 'gm')` compiles patterns safely
- `TransitionChange._checkTaskCompletionForArtifact` at transition-change.ts:348 is the reference implementation
- Pattern: `incompleteCount = content.match(incompleteRe)?.length ?? 0`, same for `completePattern`
- CLI `renderDag` at status.ts:344 draws each DAG node with `[hasTasks]` tag at line 353
