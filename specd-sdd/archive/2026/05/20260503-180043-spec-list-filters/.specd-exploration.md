## Problem statement

Add workspace filtering and an optional graph-backed search mode to the `specd specs list` command so users can restrict results to a workspace (or set of workspaces) and use the code-graph for richer text search. This improves discoverability and reduces noise when working in large monorepos.

## Approach / solution outline

- Extend the CLI `specs list` subcommand with flags:
  - `--workspace <name>` (repeatable) to filter results by workspace
  - `--query <text>` to enable a graph-backed search mode that reuses `specd graph search` internals
  - `--graph` (optional) to force graph search vs filesystem listing
- Prefer reusing the existing `ListSpecs` use-case in `@specd/core` where possible; add optional filter parameters (workspace prefix, query) to the use-case API.
- For graph-backed search, call `specd graph search "<query>" --specs --format toon` and merge/score results into the `specs list` output.
