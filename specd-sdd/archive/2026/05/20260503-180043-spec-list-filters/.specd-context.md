## LLM-Optimized Agent Instruction Block — specd

### MUST / NEVER

**Global Workflow & Spec Rules**

- NEVER write directly to `specs/`; all changes MUST follow specd workflow: draft spec → implement → review → approve → merge → archive
- `specs/` are sole source of truth; supersede code comments/external docs. MUST keep specs up to date.
- MUST read existing specs before drafting new ones; avoid duplicates, update existing specs
- MUST NOT write code without a spec; code implements specs
- MUST use specd context features for relevant specs (e.g., `default:*` for global specs)
- MUST write all artifacts, code, comments, docs in English

**Tech Stack & Architecture**

- MUST use TypeScript, pnpm monorepo, ESM only (`"type": "module"`, `NodeNext` resolution)
- Hexagonal architecture mandated:
  - Domain: no I/O, enforces own invariants; stateless services are pure functions
  - Application: ports only, no infrastructure imports
  - Infrastructure: `fs` adapters (v1 only, sole storage adapter)
- Manual DI, no IoC containers; no module-level singletons
- Layer boundaries (NEVER):
  - `domain/` → no imports from `application/`, `infrastructure/`, `composition/`
  - `application/` → no imports from `infrastructure/`, `composition/`
  - `infrastructure/` → no imports from `composition/`
- Delivery mechanisms (CLI, MCP, plugins) → only use-case factories/kernel, never ports/infrastructure
- No circular package dependencies: `plugin-*` → `skills` → `core`; `cli`/`mcp`/`code-graph` → `core`; `schema-*` → no specd deps

**Spec Layout & IDs**

- `specs/_global/` bind ALL packages; `specs/<package>/` (short names: `core`, `cli`, `code-graph`, `mcp`, `skills`, `schema-std`) bind only that package
- Every spec subdir: kebab-case, contains `spec.md` (requirements only, no WHEN/THEN) + `verify.md` (scenarios only, no prose) + mandatory `## Purpose`, `## Requirements`, `## Spec Dependencies` sections
- Spec ID format: `workspace:capabilityPath` (`:` only separator, workspace matches `/^[a-z][a-z0-9-]*$/`, `default` reserved)
- Delta files (`.delta.yaml`) MUST live in change dirs, NEVER in `specs/`
- MUST NOT add `.specd/metadata/` to `.gitignore`; commit to VCS

**CLI Rules**

- Config discovery: walk CWD to git root for `specd.yaml`; fail exit 1 if not found. `--config <path>` skips discovery, fail exit 1 if invalid.
- Output: stdout = normal, stderr = errors/warnings/diagnostics. `--format` NEVER affects stderr; errors always plain text on stderr.
- Exit codes: 0=success, 1=user/domain error, 2=hook failure, 3=system error
- Error formatting: exit 1 → stderr `error:` prefix; exit 3 → stderr `fatal:` prefix, stack trace only if `Logger.isLevelEnabled('debug')`; exit 2 → forward hook output verbatim
- `--format` flag: all commands accept `text|json|toon`, default `text`. `json`/`toon` output structured errors for `SpecdError` subtypes: `{ "result": "error", "code": "<CODE>", "message": "<MSG>", "exitCode": 1 }`
- Every leaf command MUST call `.allowExcessArguments(false)`; reject extras exit 1
- `specd --help` MUST show ASCII banner; subcommand help MUST NOT
- `specd` (no subcommand) with valid config → auto-run `project dashboard`; no config → show help
- `specd init` MUST delegate to `specd project init` identically
- Countable CLI resources: canonical plural groups (`changes`, `specs`, `archives`, `drafts`) with singular aliases (`change`→`changes`, etc.), equivalent behavior, backward compatible
- `specd specs search <query>`: required query, `--workspace` repeatable, `--graph` opt-in (unavailable → exit 1), fallback to core `SearchSpecs` with stderr warning. Search covers title, description, `spec.md` body. CLI MUST NOT parse Markdown for summary extraction.

**Core Rules**

- **Workspace**: `default` workspace mandatory, names match `/^[a-z][a-z0-9-]*$/`. `readOnly` workspaces reject writes (add spec, archive, save) with `ReadOnlyWorkspaceError`. Spec IDs internally fully-qualified (`workspace:capabilityPath`), bare paths = `default:path`. Use cases receive `ReadonlyMap<string, SpecRepository>` keyed by workspace name.
- **Change Model**: `name`/`createdAt` immutable. Lifecycle transitions only per state diagram; `implementing → verifying` only when all tasks complete. History append-only, state derived from last `transitioned` event. Artifacts: `markComplete` only via `ValidateArtifacts`, `markSkipped` only via skip use case. `skipped` only for `optional: true` artifacts, satisfies dependencies. Schema: `schemaName` mismatch → `SchemaMismatchError`; `schemaVersion` mismatch → warning only.
- **Storage**: v1 `fs` adapter only. Change dir format: `YYYYMMDD-HHmmss-<name>`. Artifact status derived at load time (not stored): `validatedHash === "__skipped__"` → `skipped`; absent file → `missing`; hash match → `complete`; else `in-progress`. Archive pattern supports `{{year}}`, `{{month}}`, `{{day}}`, `{{change.name}}`, `{{change.archivedName}}`; maintains `index.jsonl` (oldest-first). Atomic writes for manifest/index files.
- **Spec Repository Port**: interface at `packages/core/src/domain/ports/spec-repository-port.ts`, methods: `findById`, `findByIds`, `findAll`, `save`, `delete`. Throws `SpecNotFoundError` for unknown IDs. No infrastructure imports.
- **Search Specs**: `SearchSpecs.execute()` iterates workspaces in config order, merges results sorted by score descending. `includeSummary: true` adds `summary` (metadata description → spec.md fallback). Workspace errors silently caught, return other results.
- **List Specs**: `ListSpecs.execute()` iterates workspaces in config order, title: metadata → last path segment. `includeSummary`/`includeMetadataStatus` opt-in, optional fields only when requested. Errors never propagate.
- **Spec Metadata**: stored at `.specd/metadata/<workspace>/<capability-path>/metadata.json`, validated against `strictSpecMetadataSchema` before write. MUST access via `SpecRepository.metadata()`/`saveMetadata()`, never generic `artifact()`/`save()`. `dependsOn` overwrite protection (throw `DependsOnOverwriteError` unless `force`). Staleness advisory only, never block operations. Generated via `ArchiveChange` at archive time.
- **Extractor Engine**: operates on `SelectorNode`, zero I/O. Inject transforms via registry; `SubtreeRenderer` provided by caller. `from` and `childSelector` mutually exclusive, `childSelector` wins. Transform MUST return non-null string, failure → `ExtractorTransformError`.

**Code Style & Lint**

- TypeScript: `strict: true`, `noUncheckedIndexedAccess: true`, `exactOptionalPropertyTypes: true`. No `@ts-ignore`/`@ts-expect-error` without co-located why-comment.
- No `any`; use `unknown` + guards. No default exports, named exports only.
- File naming: `kebab-case.ts` for source, `.spec.ts` for tests in `test/` mirroring `src/`, no co-location. No `index.ts` barrels except package root `src/index.ts`.
- Exported functions/methods: explicit return types. Errors extend `SpecdError` with `code` string.
- Private backing fields: leading underscore (`_field`).
- Lazy loading: `list()`/`search()` return metadata only, content via explicit `get()`/`artifact()`.
- Vitest only, no Jest. No snapshot tests. Unit tests: no real I/O, port mocks fully implement interface. Integration tests: real temp dirs (`os.tmpdir()` + unique subfolder). Test naming: `"given <state>, when <action>, then <outcome>"`.

**Logging**

- Logging interface compatible with `console` API: `log()` (aliases `info()`), `info()`, `debug()`, `warn()`, `error()`. Extended levels: `fatal` → `console.error()` + `[FATAL]`, `trace` → `console.debug()` + `[TRACE]`.
- MUST NOT use global `console` directly in production code; use project logging abstraction.

**Docs**

- All docs under `docs/` (adr/, cli/, mcp/, core/, schemas/). Only `README.md`, `AGENTS.md`, `CLAUDE.md` outside `docs/`.
- ADRs: MADR format, `NNNN-kebab-case-title.md`, required sections: `## Context and Problem Statement`, `## Decision Drivers`, `## Considered Options`, `## Decision Outcome` (with `### Spec` linking specs), `## Pros and Cons of the Options`. MUST link specs, never depend on ADRs directly.
- CLI/MCP/Core docs: every command/tool/port/use case MUST have docs with purpose, flags, examples, exit codes, contracts.
- JSDoc: all functions, methods, classes, types MUST have JSDoc with description, `@param`, `@returns` (omit for void), `@throws`.

### SHOULD

- Include `// comment` in CLI help schema only when field name not self-explanatory.
- Prefer kernel (`createKernel`) for grouped use case access.
- LLM MAY improve metadata via `SaveSpecMetadata` (refine `description`, `keywords`).
- Use `resolveSpecPath` transform: tries value first, then interpolated args, returns first canonical spec ID.
- Declare `prefix` for workspace logical grouping, `contextIncludeSpecs`/`contextExcludeSpecs` for context scope.
- Omit empty ADR sections (except `### Spec` in `## More Information`).
- Prefer `readonly` on domain objects, `as const` for literal configs.
