# Exploration: invalidate-drift-poll-dedupe

## Problem statement

SpecD Studio polls project/change status every ~2.5s (`useProjectPoll` in `packages/ui/src/hooks/use-project-poll.ts`). Each status read loads the active change via `ChangeRepository.get()`, which runs `_reconcileArtifactDrift`. When validated artifacts still have drift, the repository called `change.invalidate('artifact-drift', ...)` and rewrote `manifest.json` on **every** load — even when the change was already invalidated and in `designing`. Users saw constant re-invalidation and manifest churn.

## Approach / solution outline

1. **Domain dedupe (`Change.invalidate`)**: For `cause === 'artifact-drift'` only, if the change is already in `designing` and the last `invalidated` event has the same cause and the same expanded affected-artifact set (normalized), skip appending a new `invalidated` / `transitioned` event. Still mark drift flags on focused files when applicable.
2. **Repository guard (`FsChangeRepository._reconcileArtifactDrift`)**: Only persist manifest when `change.updatedAt` actually changed after `invalidate()` (i.e. a new event was recorded).

Manual invalidation (`artifact-review-required`, overlap, etc.) must **not** be deduped — first implementation briefly deduped all causes and broke `GetStatus` overlap-detail tests; fixed by scoping dedupe to `artifact-drift` only.

## Design decisions (to encode in deltas)

- Dedupe applies **only** to automatic `artifact-drift` invalidation, not manual CLI/UI invalidation or overlap conflicts.
- Equivalence of affected sets: normalize by artifact type + sorted file keys (after policy expansion).
- When deduped, drift flags (`hasDrift` / `markDrifted`) may still be applied on focused files.
- Repository must not write manifest when invalidation was a no-op (`updatedAt` unchanged).
- Parallels existing rule in `core:validate-artifacts`: at most one `artifact-drift` invalidate per validate execution.

## Rejected alternatives

- Disabling drift reconciliation on read-only status polls — would hide real drift until another mutation.
- Broad dedupe for all invalidation causes — breaks merging multiple overlap invalidation events in `GetStatus` review projection.
