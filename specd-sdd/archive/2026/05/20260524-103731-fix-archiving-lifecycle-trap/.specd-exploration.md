# Exploration Context

## Problem Statement

The user reported a lifecycle state machine problem: once a change enters `archiving`, it cannot transition to any other state (`archivable`, `designing`, `implementing`, etc.). Archive can fail at multiple points (hooks, overlap, preflight, publish), leaving the change stuck with no recovery path except retrying `spec_item archive`.

Root cause confirmed in code:

```typescript
// packages/core/src/domain/value-objects/change-state.ts
archiving: [],  // dead end — no valid transitions
```

Additionally, `archivable → archiving` happens at the **very start** of `ArchiveChange.execute()` (first `ChangeRepository.mutate`), **before**:

- overlap guard
- readOnly guard
- pre-archive hooks
- prepare + preflight

So a failing pre-hook (e.g. `pnpm test`) leaves the change in `archiving` without modifying any files — contradicting the intent that pre-commit failures should not trap the workflow.

This was previously
flagged in issue `#55` exploration (`harden-archive-reload-consistency`) as open question: _"If archive cannot move the archiving transition later, should it explicitly roll back state on pre-merge failures?"_

## Approach / Solution Outline

Two coordinated fixes (user confirmed both):

### 1. Defer `archivable → archiving` transition

Move the persisted state transition to **immediately before canonical publication**, after:

- schema name guard
- `assertArchivable()` (check only, no transition)
- overlap guard
- readOnly guard
- pre-archive hooks
- full-batch prepare + preflight

New order:

```
archivable → [guards + hooks + preflight in archivable] → mutate: transition to archiving → publish → metadata → move to archive → post-hooks
```

Rationale:

- Pre-hook failures, overlap blocks, readOnly errors, and preflight failures leave the change in `archivable` — clean state with normal transitions available.
- The `serialize-change-mutations` rationale (April 2026) for early transition was concurrency, not lifecycle semantics. A single `mutate` just before publish still serializes the commit boundary without trapping on hook failures.
- `RunStepHooks` uses workflow step `'archiving'`, not lifecycle state — pre-hooks can run while still in `archivable`.

### 2. Add escape transitions from `archiving`

Extend `VALID_TRANSITIONS`:

```typescript
archiving: ['archivable', 'designing'],
```

Behavior:

- **`archiving → archivable`**: auto-rollback on pre-commit failures (`commitStarted: false` in `archive-failed` events), including failures that today don't even record `archive-failed` (overlap, readOnly, hooks — though those should no longer reach `archiving` after fix #1).
- **`archiving → designing`**: manual escape via `specd change transition <name> designing` when the failure requires spec/delta revision; aligns with domain rule _"Every state except drafting MAY return to designing"_ — `archiving` is currently the exception.
- **Do NOT** add transitions to `implementing`, `verifying`, `done` from `archiving` — path should be `archiving → designing → …`.

Optional enhancement during implementation: auto-rollback to `archivable` inside `_recordArchiveFailure` when `commitStarted === false`.

## Design Decisions

- User explicitly confirmed both fixes: new transitions AND deferred transition timing.
- Transition should happen just before `publish`, not before hooks.
- Escape to `designing` is manual; escape to `archivable` can be automatic on pre-commit failures.
- Do not allow arbitrary backward jumps to `implementing`/`verifying`/`done` from `archiving`.
- Pre-hooks remain runnable while in `archivable` state (workflow step name unchanged).

## Rejected Alternatives

- **Only add escape transitions without moving transition timing** — insufficient; hooks would still trap in `archiving`.
- **Only move transition timing without escape transitions** — insufficient; post-publish failures still need manual recovery.
- **Allow `archiving → implementing/verifying/done`** — rejected; confusing semantics; use `designing` as hub.
- **Move transition after hooks but before preflight only** — user specifically said "justo antes de publish"; preflight is also pre-commit and should stay in `archivable`.
