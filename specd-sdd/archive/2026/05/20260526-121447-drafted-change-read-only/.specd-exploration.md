# Exploration: drafted-change-read-only

## Problem statement

Drafted changes (stored under `.specd/drafts/`) are currently treated like active changes at the core layer: `ChangeRepository.get()` and `mutate()` resolve by name across `changes/` and `drafts/` with no immutability check. Any use case that calls `mutate()` — `TransitionChange`, `ValidateArtifacts`, `EditChange`, approvals, etc. — can modify a drafted change as if it were in progress. That breaks the shelf semantics and risks corrupting workflow state.

## Approach / solution outline

Enforce **read-only for drafted changes in core**, not only in CLI:

1. **Repository split** — `get`/`mutate` active-only; `getDraft`/`mutateDraft` for `drafts/`; `listDrafts` → `DraftedChangeView[]`.
2. **Read model** — `DraftedChangeView` facade (no mutable `Change` escape hatch).
3. **Secondary guard** — `DraftedChangeReadOnlyError` on `save`/`saveArtifact` outside `mutateDraft`.
4. **Allowlist while drafted** — `RestoreChange`, `DiscardChange` via `mutateDraft`; reads via `getDraft` / `GetDraft` / read-only `GetStatus`.
5. **Specs** — extend `core:change` with a requirement that drafted changes MUST NOT accept lifecycle transitions, artifact validation persistence, scope edits, approvals, or other transforming operations until restored.
6. **Port contract** — document guard behavior on `core:change-repository-port` (`mutate` semantics).
7. **CLI** (likely follow-up in design) — surface `DraftedChangeReadOnlyError` with clear messages; existing `drafts show` already checks `isDrafted` but is not a domain guard.

Optional API pattern discussed: `mutate(name, fn, { allowDrafted: true })` only for restore/discard internals — design phase should pick one approach and keep a single enforcement door.

  override async mutate<T>(name: string, fn: (change: Change) => Promise<T> | T): Promise<T> {
    return this._withChangeLock(name, async () => {
      const change = await this.get(name)
      if (change === null) {
        throw new ChangeNotFoundError(name)
      }
      const result = await fn(change)
      await this.save(change)
      return result
    })
  }
