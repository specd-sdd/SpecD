# Exploration Context — spec-lock-sidecar

Generated: 2026-05-12

## Problem statement

Issue: https://github.com/specd-sdd/SpecD/issues/56

Archived specs currently lose durable schema identity and durable spec dependency (`dependsOn`) data at the spec level. The issue proposes an immutable sidecar (`spec-lock.json`) written at archive time to preserve:

- schema name/version from the change creation context
- dependsOn from `change.specDependsOn`

The issue also requires conflict detection when metadata extraction yields `dependsOn` that contradict sidecar values.

## Approach / solution outline discussed

- Introduce `spec-lock.json` as infrastructure sidecar, written during archive/re-archive.
- Treat sidecar as source of truth for schema identity and persisted dependsOn.
- Merge/update `dependsOn` across re-archives for the same spec.
- Keep behavior independent of metadata extraction availability.
- Validate mismatch between extraction-derived dependsOn and sidecar dependsOn as failure.
