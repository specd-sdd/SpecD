# Exploration Context

## Problem statement

The user asked to implement GitHub issue `#16` for `specd-sdd/SpecD`.

Issue URL:

- `https://github.com/specd-sdd/SpecD/issues/16`

The issue states that `specd.local.yaml` currently behaves as a full replacement for `specd.yaml`. When the local file exists, `specd.yaml` is not read at all. This makes local customization expensive because users must duplicate the full config even for small changes.

The requested outcome is a layered config cascade for `specd.yaml`, named variants, and local variants, while preserving current standalone-local behavior when `specd.local.yaml` does not opt into cascading.

## Approach / solution outline

The issue proposes a presence-based cascade with no explicit activation flag:

Default discovery order:

1. `specd.yaml`
2. `specd.*.yaml` in alphabetical order
3. `specd.local.yaml`
4. `specd.local.*.yaml` in alphabetical order

Each later file merges on top of the previously accumulated config.

Standalone-local fallback:

- If `specd.local.yaml` does not opt into cascade via `extends` or equivalent, it remains a self-contained config base.
- In that mode, the cascade starts at `specd.local.yaml` and continues only with `specd.local.*.yaml`.

Explicit file selection:

- `--config <file>` must bypass discovery and presence-based cascade.
- Only the selected file and its own `extends` chain should be used.

Multiple explicit files:

- The issue proposes allowing repeated `--config` flags and merging those files in declaration order.

Merge semantics requested by the issue:

- Scalars: last layer wins
- Objects: deep merge by key
- New keys: additive
- Arrays: additive append-by-default
- Array removals: `remove` block using YAML AST selectors compatible with the existing selector model already used by delta validation infrastructure

Optional `id` fields are proposed on removable array items:

- context entries
- workflow hook entries
- plugins

Gitignore convention requested by the issue:

- committed: `specd.yaml`, `specd.*.yaml`
- gitignored: `specd.local.yaml`, `specd.local.*.yaml`
