## Key Technical Findings from Investigation

1. Markdown adapter node vocabulary includes:
   - `document`, `section`, `paragraph`, `list`, `list-item`, `code-block`, `thematic-break`
   - Source anchor: `packages/core/src/infrastructure/artifact-parser/markdown-parser.ts` (`nodeTypes()`).
2. Selector matching is generic by `type` and supports:
   - `matches`, `contains`, `parent`, `level`, `where`, `index`
   - Source anchor: `packages/core/src/domain/services/selector-matching.ts` (`nodeMatches`).
3. Existing tests already prove non-section selector behavior, e.g.:
   - paragraph remove with `contains`
   - code-block remove with `matches`
   - Source anchor: `packages/core/test/infrastructure/artifact-parser/markdown-parser.spec.ts`.
4. Delta spec contract states that ambiguous selector matches in `modified`/`removed` must fail with `DeltaApplicationError`.
5. Potential practical failure mode for paragraph replacement:
   - Selector ambiguity when multiple paragraph nodes match a broad `contains` expression and missing `parent` scoping.
