The user wants to add a new public-facing website for `specd` using Docusaurus.

## Design Decisions Reached

- The site should live in a new workspace, not be folded into `core` or `cli`.
- The workspace should be named `public-web`.
- The implementation path should be `apps/public-web`.
- The spec path should be `specs/public-web`.
- The site should be built with Docusaurus.
- ADRs should not be included in the public site.
- The public home should be a presentation/marketing-style page, not a default docs index.
- The configuration was updated before change creation so `public-web` now exists as an owned workspace in `specd.yaml`.
- The change is being created with no initial specs so that `public-web:*` specs can be added cleanly during design.

## Agreements Reached

- This initiative should go through normal `specd` workflow with specs.
- A new workspace was the right model for this area of the repo.
- The agent should not try to invent `public-web:*` specs before the workspace exists in config.
- Creating the change with no initial specs is acceptable and supported by `specd-new`.

- Forcing specs into the change before the workspace existed:
  Rejected as a bad fit for the workflow and config model.
