Generated: 2026-04-23

## Problem statement

The current specd entry skill (`/specd`) runs 6 CLI commands at startup to gather project state:

- `config show --format json`
- `spec list --format json --summary`
- `change list --format json`
- `drafts list --format json`
- `project context --format text`
- `graph stats --format json`

This is slow and verbose. The user wants a single `project status` command that consolidates this information.

## Proposed solution

Create `specd project status` command that combines:

1. `project dashboard` output (workspaces, specs, changes)
2. `graph stats` output (freshness, indexing stats)
3. `project context` references (which specs/files to read, not content)

Flags:

- `--context` to include project context references (without loading file content)
- `--graph` to include graph stats

The command should be simple, JSON output, no "florituras".

## Agreements reached

- The new command should replace the 6-command startup sequence in the specd skill
- `project context` should show references only (file paths/spec IDs), not full content
- Output should be minimal and machine-readable

## Open questions

- Should `--context` flag be default on or off?
- Should the command also run `graph index` if stale?
