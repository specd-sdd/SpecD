# Exploration: project-status-workspace-details

## Problem Statement

The `project status` command currently shows limited information about workspaces. Users and agents need to know whether a workspace is external to the current repository (`isExternal`) and where its code is located (`codeRoot`) to understand the project structure and context locality.

## Intent

The user requested: "En project status los workspaces deben mostrar isExternal y codeRoot".

## Approach

- Modify `cli:cli/project-status` spec to include `isExternal` and `codeRoot` requirements for workspaces.
- Update `packages/cli/src/commands/project/status.ts` to implement these requirements.
- Ensure the information is available in both text and structured (JSON/Toon) formats.

## Affected Areas

- **Spec:** `cli:cli/project-status`
- **Code:** `packages/cli/src/commands/project/status.ts`
- **Domain:** `SpecdWorkspaceConfig` already contains these fields, so no changes are needed in `packages/core`.

## Key Codebase Observations

- `packages/cli/src/commands/project/status.ts` uses `config.workspaces` which contains `SpecdWorkspaceConfig`.
- `SpecdWorkspaceConfig` (in `packages/core/src/application/specd-config.ts`) has `codeRoot: string` and `isExternal: boolean`.
- Non-text formats (`json`, `toon`) already include `codeRoot` but missing `isExternal`.
- Text format is missing both or has them poorly integrated.

## Design Decisions

- Workspaces in text format should show: `name (prefix: prefix) [ownership, isExternal, codeRoot: path]`.
- Workspaces in JSON/Toon format should include `isExternal` boolean.

## Agreements

- The change follows the `specd` workflow strictly.
- English is used for all artifacts and code.
