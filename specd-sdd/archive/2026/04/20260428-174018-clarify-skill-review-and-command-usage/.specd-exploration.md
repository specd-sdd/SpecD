# Exploration Context

## Problem statement

The user requested a new change to improve specd skills so they reflect recent CLI command evolution and reduce unnecessary multi-command workflows.

Primary pain points raised by the user:

- Skills still mention older command shapes/parameters in places where newer options can collapse 2-4 calls into one.
- The workflow message around `changes validate` is not explicit enough: users may interpret it as full content review, when it is structural/state validation.
- Review guidance must explicitly require semantic/content review, not only passing validation.
- Overlap drift can leave deltas obsolete; before applying or trusting deltas, merged content must be checked with `changes spec-preview` to avoid overwriting important material.

## Approach / solution outline

Create a dedicated change focused on workflow guidance and specs tied to command usage and validation/review semantics.

Scope intent:

- Update workflow automation/skills guidance to prefer current canonical commands and newer flags where they replace multi-step command sequences.
- Clarify the boundary between structural validation (`changes validate`) and human/agent content review.
- Add explicit anti-drift/overlap safety guidance requiring `changes spec-preview` checks when deltas might be stale.
