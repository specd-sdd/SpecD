# Exploration Context: fix-metadata-workspace-prefix

**Generated:** 2026-04-18

## Problem Statement

When generating metadata for specs in workspaces without a `prefix` configured (like `plugin-manager`, `skills`, `plugin-agent-*`), the code only adds the prefix to the spec path, but does NOT add the workspace name. This causes metadata to be stored at `.specd/metadata/<prefix>/<spec>` instead of `.specd/metadata/<workspace>/<prefix>/<spec>`.

For workspaces WITH `prefix` (like `core`, `cli`, `default`), this works correctly because the prefix equals the workspace name. But for workspaces WITHOUT `prefix`, the workspace name is missing entirely.

## Root Cause

In `packages/core/src/composition/kernel-internals.ts`, the `metadataPath` is computed as:

```typescript
metadataPath = path.join(vcsRoot, '.specd', 'metadata')
```

This is shared across ALL workspaces. The `FsSpecRepository` then uses its `prefix` config to determine where to store metadata, but the workspace name is never added to the path.

The `_metadataFilePath` method in `spec-repository.ts` uses:

```typescript
return path.join(this._metadataPath, name.toFsPath(path.sep), 'metadata.json')
```

Where `name` includes the prefix segments but NOT the workspace.

## Affected Areas

- **Package:** `@specd/core`
- **File:** `packages/core/src/infrastructure/fs/spec-repository.ts` (the `FsSpecRepository` class)
- **File:** `packages/core/src/composition/kernel-internals.ts` (where metadataPath is computed)
- **Spec:** `core:core/spec-metadata` (defines expected behavior)

## Workspaces Affected

Workspaces WITHOUT `prefix` in `specd.yaml`:

- `skills` (no prefix)
- `plugins` (has prefix)
- `plugin-manager` (no prefix)
- `plugin-agent-claude` (no prefix)
- `plugin-agent-copilot` (no prefix)
- `plugin-agent-codex` (no prefix)

## Solution Approach

The fix requires modifying how `metadataPath` is computed in `kernel-internals.ts` to include the workspace name:

```typescript
// Current: metadataPath = path.join(vcsRoot, '.specd', 'metadata')
// Fixed:   metadataPath = path.join(vcsRoot, '.specd', 'metadata', ws.name)
```

This ensures each workspace has its own metadata subdirectory: `.specd/metadata/core/`, `.specd/metadata/plugin-manager/`, etc.
