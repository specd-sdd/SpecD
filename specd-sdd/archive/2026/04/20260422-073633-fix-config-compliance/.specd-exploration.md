# Exploration: fix-config-compliance

## Problem statement

The `specd.yaml` config loader and related infrastructure have multiple compliance issues against `specs/core/config/spec.md`. A compliance report (`specs-compliance-config-20260422-081822.md`) found 75% conformant (15/20), 3 divergent, and 2 not implemented requirements. Additionally, during investigation the user identified that `artifactRules` and `workflow` fields still exist in code despite specs saying they were replaced by `schemaOverrides`.

## Approach / solution outline

1. **Remove stale fields** — `artifactRules` from config Zod schema, SpecdConfig interface, CLI show command, and docs
2. **Remove `workflow`** if it exists as a top-level config field (check during design)
3. **Add `skills` manifest** — parse `skills` section in `specd.yaml` through the Zod schema and expose on `SpecdConfig` (R17)
4. **Add `plugins` validation** — bring `plugins` into the Zod schema so ConfigLoader validates it instead of ConfigWriter bypassing validation (R18)
5. **Align ConfigWriter port** — update `addPlugin` signature in spec or code to match (R19)
