# Exploration: specd spec show --artifact

## Problem Statement

Users and scripts need to extract specific artifact content (e.g., just `spec.md` or just `verify.md`) from a spec without having to manually parse the combined output of `specd spec show`.

## Approach / Solution Outline

Add an `--artifact <name>` flag to the `specd spec show` command.

- The `<name>` should correspond to the artifact ID defined in the schema (e.g., `specs`, `verify`).
- The command must resolve this ID to the actual filename using the schema's `output` pattern.
- For `@specd/schema-std`:
  - `specs` -> `spec.md`
  - `verify` -> `verify.md`
- If the flag is provided, only that specific artifact should be printed (if it exists).
- Should work with all formats (`text`, `json`, `toon`).

## Affected Areas

- **Spec:** `cli:cli/spec-show` (needs update)
- **Code:** CLI implementation of `spec show` (likely in `packages/cli/src/commands/spec/show.ts` or similar).

## Design Decisions

- The flag name is `--artifact`.
- It must resolve via schema to be future-proof for custom schemas.

## Key Codebase Observations

- Schema `@specd/schema-std` defines:
  - `specs` with `output=specs/**/spec.md`
  - `verify` with `output=specs/**/verify.md`
- `specd schema show` reveals these mappings.
