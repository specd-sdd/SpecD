# Exploration Context: add-specs-outline-command

## Problem Statement

The user wants a CLI command to retrieve the "outline" (navigable structure) of a spec artifact. This is useful for understanding the structure of a spec without reading its full content, especially for AI agents or for quick inspection.

## Approach / Solution Outline

- Add `specd specs outline <specPath>` command to `@specd/cli`.
- Implement `GetSpecOutline` use case in `@specd/core`.
- Support `--artifact <id>` flag: resolve artifact ID to filename using the active schema.
- Support `--file <name>` flag: specify a filename directly within the spec directory.
- Use `ArtifactParser.outline(ast)` domain method to generate the outline.
- Support `text`, `json`, and `toon` output formats.

## Affected Areas

- `packages/cli/src/commands/spec/outline.ts`: New command implementation.
- `packages/cli/src/index.ts`: Register the new command.
- `packages/core/src/application/use-cases/get-spec-outline.ts`: New use case.
- `packages/core/src/composition/kernel.ts`: Wire and expose the new use case.
- `specs/cli/spec-outline/`: New spec.
- `specs/core/get-spec-outline/`: New spec.

## Design Decisions

- If `--artifact` is provided, the command should look up the artifact in the active schema to find its `output` filename.
- The `outline` should be rendered hierarchically in `text` format (using indentation or prefixes like `#`, `##`).
- `json`/`toon` format should return an array of `OutlineEntry` objects.

## Key Codebase Observations

- `ArtifactParser` port already defines an `outline(ast)` method.
- `MarkdownParser`, `JsonParser`, `YamlParser`, and `PlaintextParser` already implement `outline`.
- `specd specs show` implementation in `packages/cli/src/commands/spec/show.ts` provides a good reference for artifact resolution.
- `GetArtifactInstruction` use case already uses `parser.outline(ast)` for delta context.
