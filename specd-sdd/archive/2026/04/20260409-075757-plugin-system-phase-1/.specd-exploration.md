## Key issue context captured from GitHub

Issue title:
`Redesign plugin system: autonomous agent plugins, canonical skill prompts, shared helpers`

Concrete problem statements captured from the issue body:

- The CLI currently hardcodes a `KNOWN_AGENTS` map in 5 separate files and only
  actually supports `claude`.
- The interactive wizard in `project init` offers `copilot` and `codex` but silently
  drops them.
- Plugin packages `plugin-claude`, `plugin-copilot`, and `plugin-codex` are empty
  stubs with no defined interface.
- `@specd/skills` currently exports a flat `listSkills()` / `getSkill()` API returning
  `{ name, description, content }` without prompt typing, versioning, or plugin-facing
  structure.
- The CLI currently owns agent-specific file-writing logic that should belong to the
  plugin.
- Common file operations are duplicated across commands.

Proposed redesign areas captured from the issue:

- `@specd/skills` becomes the canonical prompt library with:
  - typed prompts
  - versioned prompts
  - shared helpers
- Plugins become autonomous, with a base `SpecdPlugin` interface and type-specific
  interfaces such as `AgentPlugin`.
- Config and CLI become orchestration only.
- Existing CLI `skills` commands are replaced by `plugins` commands.
- `project init` and `project update` share the same plugin-install/update logic.
