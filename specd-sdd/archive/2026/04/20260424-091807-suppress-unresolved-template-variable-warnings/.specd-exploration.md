# Exploration Context

## Problem Statement

The user wants to remove console warning output that appears when template variables like `{{variable}}` cannot be resolved during template expansion in hooks. The warning currently appears as "warning: unresolved template variable '{{token}}'" on the console.

## Approach / Solution Outline

Remove the console.warn callback in kernel-internals.ts that logs warnings for unresolved template variables. The TemplateExpänder class has an optional `onUnknown` callback that gets invoked when tokens can't be resolved - currently it's wired to console.warn.

## Affected Areas

- **Code:** `packages/core/src/composition/kernel-internals.ts` (line 474-476)
- **Class:** `TemplateExpander` in `packages/core/src/application/template-expander.ts`

## Key Codebase Observations

The warning is created in kernel-internals.ts:

```typescript
const expander = new TemplateExpander({ project: { root: config.projectRoot } }, (token) => {
  console.warn(`warning: unresolved template variable '{{${token}}}'`)
})
```

The TemplateExpander class accepts an optional `onUnknown` callback that is called when a token cannot be resolved (see template-expander.ts:123-126). The solution is to pass `undefined` instead of a callback, or simply not invoke `this._onUnknown` at all.
