# Problem statement

The user wants to reduce noise and token volume in change instructions while improving delta authoring reliability for LLMs.

Current pain points discussed:

- Change instructions currently include outlines for ALL specs in a change, which creates excessive output and makes content hard to navigate.
- Outline generation appears incomplete (only some types are shown), which may prevent the LLM from understanding all valid selector targets when generating deltas.

Desired behavior:

- Change instructions should expose only a compact list of available outlines, specifically in the form:
  - availableOutlines: ['core:core/config', 'cli:otro-spec']
- Full outline content should be fetched only on demand using:
  - specs outline --artifact <artifact>
- Outline generation should include all types and fields supported by delta selectors.
- Skills/instructions should be updated so agents use specs outline command on demand instead of relying on massive embedded outlines.

# Approach / solution outline

Three-part change:

1. Compact output in change artifact instructions:

- Replace embedded multi-spec outlines with an `availableOutlines` list of spec IDs.

2. Expand outline model coverage:

- Ensure `specs outline` includes all selector-relevant node types and fields supported by the delta selector model.

3. Skill workflow updates:

- Update relevant skill guidance to fetch outlines via `specs outline --artifact <artifact>` when needed.
