## Problem statement

The user wants to add a new agent plugin package (`@specd/plugin-agent-standard`) that targets the [Agent Skills open standard](https://agentskills.io/). This provides a vendor-neutral installation path for specd skills, complementing the existing vendor-specific plugins (claude, copilot, codex, opencode). The plugin installs skills into `.agents/skills/` using the standard's frontmatter format, including the `allowed-tools` field for pre-approved tool access.

## Key codebase observations

### Agent Skills standard (agentskills.io/specification)

Frontmatter fields:

- `name` (required) — max 64 chars, lowercase + hyphens
- `description` (required) — max 1024 chars
- `license` (optional)
- `compatibility` (optional) — max 500 chars, environment requirements
- `metadata` (optional) — arbitrary key-value map
- `allowed-tools` (optional, experimental) — space-separated string of pre-approved tools

### Existing plugin pattern (from opencode)

Each plugin has identical structure:

- `src/index.ts` — factory with `create()` export, reads manifest
- `src/domain/types/<name>-plugin.ts` — class implementing `AgentPlugin` from `@specd/plugin-manager`
- `src/domain/types/frontmatter.ts` — interface for supported frontmatter fields
- `src/domain/frontmatter/index.ts` — per-skill frontmatter map (keyed by skill name)
- `src/application/use-cases/install-skills.ts` — installs skills to target dir, prepends frontmatter
- `src/application/use-cases/uninstall-skills.ts` — removes skill dirs
- `test/install-skills.spec.ts` — integration test with temp directory

### Key difference between opencode and claude frontmatter

- OpenCode: `{ name, description, license, compatibility, metadata }` — standard fields only
- Claude: `{ name, description, allowed_tools, argument_hint, when_to_use, disable_model_invocation, user_invocable, model, effort, context, agent, hooks, paths, shell }` — many vendor-specific fields

The new standard plugin should use the agentskills.io field set, which is closest to opencode but adds `allowed-tools`.
