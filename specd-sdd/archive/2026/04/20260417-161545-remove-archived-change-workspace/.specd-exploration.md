# Exploration Context

**Generated:** 2026-04-17

## Problem Statement

The `ArchivedChange` entity has a `workspace: SpecPath` property that is redundant — it is derived from the first `specId` at archive time. This creates an inconsistency with `Change`, which computes `workspaces` as a getter at runtime rather than storing it. Additionally, the archive index persists this unnecessary field.

## Affected Areas

- `packages/core/src/domain/entities/archived-change.ts` — entity definition
- `packages/core/src/infrastructure/fs/archive-repository.ts` — `_buildArchivedChange()`, `_buildIndexEntry()`, `deriveFirstWorkspace()`
- Archive index file `.specd-index.jsonl` — persisted workspace field

## Solution

1. Remove `workspace: SpecPath` from `ArchivedChange` entity
2. Add `workspaces` getter that derives from `specIds` (like `Change`)
3. Remove `workspace` field from archive index entries
4. Update code that reads `archivedChange.workspace` to derive from `specIds`

## Key Findings

- `Change.workspaces` is a getter derived from `specIds` (line 246 in change.ts)
- `ArchivedChange` incorrectly stores it as a persisted field
- Archive index has redundant `workspace` field that duplicates info in `specIds[0]`
- `deriveFirstWorkspace()` in archive-repository.ts extracts from manifest.specIds[0] or manifest.workspaces[0] as fallback
