## Problem Statement

The `@specd/cli` package (adapter package) contains business logic relating to the code-graph that should instead reside within the `@specd/code-graph` package. This violates the clean/hexagonal architecture constraint (see [specs/\_global/architecture/spec.md](file:///Users/monki/Documents/Proyectos/specd/specs/_global/architecture/spec.md) - "Adapter packages contain no business logic") and prevents other delivery mechanisms (MCP, public API, public-web) from reusing this logic without duplication.

## Approach / Solution Outline

Move all code-graph related business logic out of `@specd/cli` into `@specd/code-graph` (or `@specd/core` where appropriate). Simplify CLI commands to act only as orchestrators that delegate to `@specd/code-graph`'s `CodeGraphProvider` and format results.

Specific items to refactor:

1. **Multi-file Impact Aggregation:**
   - Current logic: Located in `packages/cli/src/commands/graph/impact.ts` (in `handleFilesImpact`). It loops over files, computes individual impacts, aggregates direct/indirect/transitive counts, computes total affected files, and finds the maximum risk level.
   - Target logic: Move this aggregation into `CodeGraphProvider` or a domain service in `@specd/code-graph` (e.g., `analyzeFilesImpact(filePaths: string[], direction, maxDepth)`).
2. **Index Locking/Mutex:**
   - Current logic: Managed in `packages/cli/src/commands/graph/graph-index-lock.ts` (`acquireGraphIndexLock`, `assertGraphIndexUnlocked`).
   - Target logic: Move locking management to `@specd/code-graph` or `@specd/core`.
3. **Bootstrap Configuration:**
   - Current logic: Managed in `packages/cli/src/commands/graph/bootstrap-graph-config.ts` (`createBootstrapGraphConfig`).
   - Target logic: Move fallback configuration creation to `@specd/code-graph` or `@specd/core`.
4. **Project Configuration Merging:**
   - Current logic: Merged in `packages/cli/src/commands/graph/build-project-graph-config.ts` (`buildProjectGraphConfig`).
   - Target logic: Move logic to `@specd/code-graph`.
5. **Staleness Calculation:**
   - Current logic: Located in `packages/cli/src/commands/graph/stats.ts` (`computeStaleness`).
   - Target logic: Integrate staleness calculation/check in `CodeGraphProvider` or `@specd/code-graph` services.
