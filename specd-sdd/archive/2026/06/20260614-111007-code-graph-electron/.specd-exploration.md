# Exploration: code-graph-electron

## Goal

Create a new `@specd/code-graph-electron` workspace that allows `@specd/studio-desktop`
to use the code graph locally under Electron without sharing the same native
`better-sqlite3` runtime path used by CLI and API.

## Confirmed problem

- `codegraph` works in CLI.
- `codegraph` works in API.
- `codegraph` fails only in `studio-desktop` local mode.
- The likely root cause is ABI mismatch for `better-sqlite3` when loaded from
  Electron main.

## Current desktop wiring

- Renderer mounts `SpecdApp` in embedded mode with a local desktop data adapter.
- The local adapter forwards graph calls through Electron preload IPC.
- Electron main handles graph methods in `apps/specd-studio-desktop/src/main/ipc-handlers.ts`.
- Those handlers call `createCodeGraphProvider(config)`.
- The provider currently comes from `packages/code-graph/dist/index.js`.
- `createCodeGraphProvider()` uses the default `sqlite` graph store.
- `SQLiteGraphStore` imports `better-sqlite3`.

## Constraints discussed

- Do not require developers to manage Node versions manually.
- Avoid solutions that are fragile for external contributors with different local Node versions.
- Avoid shipping a second full runtime alongside Electron if possible.
- Do not externalize the graph to a separately launched helper that users must run by hand.
- Keep desktop distributable as a single app.
- Avoid a manual fork of `packages/code-graph`.

## Alternatives discussed

### Rebuild one shared `better-sqlite3`

Rejected as the primary direction.

Reason:
- CLI/API and Electron load native modules under different runtimes.
- A single shared native resolution path is too fragile for contributors and local development.

### Helper process outside Electron

Not chosen for this change.

Reason:
- If the helper still uses Electron's runtime, the ABI problem remains.
- If it ships a separate Node runtime, packaging becomes much heavier.

### Separate Electron-specific package

Chosen direction for this change.

The package should:
- be named `@specd/code-graph-electron`
- be consumed only by `studio-desktop`
- keep `cli` and `api` on `@specd/code-graph`
- build from the shared source in `packages/code-graph/src`
- own its own effective `better-sqlite3` resolution/rebuild path for Electron

## Expected shape

- Add workspace `packages/code-graph-electron`
- Add specs workspace `specs/code-graph-electron`
- Move `studio-desktop` imports away from `packages/code-graph/dist/index.js`
- Point desktop graph composition at `@specd/code-graph-electron`
- Preserve the existing `@specd/code-graph` package for CLI and API

## Open questions carried into proposal/design

- Exact build structure for compiling from `packages/code-graph/src` without forking code
- Exact packaging and dependency-resolution strategy that guarantees Electron loads the Electron-specific native addon path rather than the standard one
