# Exploration Context: llm-optimized-metadata

Generated: 2026-06-03

## Problem statement

The specd-metadata skill currently optimizes the full metadata of a spec and attempts to overwrite the deterministic metadata file. This causes conflicts or loss of deterministic data.
The goal is to separate deterministic metadata from LLM-optimized content by adding two new fields: `optimizedDescription` and `optimizedContext` to the metadata schema. Agents will populate these fields, and tools will consume them when available.

## Approach / solution outline

1.  **Schema Update:** Add `optimizedDescription` and `optimizedContext` to `SpecMetadata` and `strictSpecMetadataSchema` in `core/src/domain/services/parse-metadata.ts`.
2.  **Invalidation:** The normal `ArchiveChange` workflow completely overwrites `metadata.json` without these fields, naturally invalidating them when a spec is modified and archived.
3.  **Persistence:** `SaveSpecMetadata` allows saving these fields since they will be part of `strictSpecMetadataSchema`.
4.  **Context Compiler:** In `get-spec-context` and `compile-context`, if `llmOptimizedContext: true` is active and the fields exist, use `optimizedContext ?? context`. If the fields are missing (e.g. after archiving), emit a warning or signal in the CLI (`change context` and `spec context`) that the metadata needs to be regenerated by the skill.
5.  **Code Graph:** The code graph indexer (`indexer`) should use `optimizedDescription ?? description` for its BM25 searches and displays.
