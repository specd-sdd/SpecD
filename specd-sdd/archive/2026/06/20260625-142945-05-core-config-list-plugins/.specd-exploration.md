# 05-core-config-list-plugins — exploration

## Source

`core-refactor-on-main.md` phase **P1c**.

## Problem statement

`ListPlugins` in `kernel.project` re-reads disk via `ConfigWriter` but only projects `config.plugins` — redundant with `SpecdConfig` already in kernel.

## Approach

1. Remove `ListPlugins` from `Kernel.project` interface and `kernel.ts`
2. CLI `plugins list`: `loadConfig` → `config.plugins` (filter by type in command)
3. Optional pure helper `listPluginDeclarations(config, type?)` in `core/application/` — may export in public barrel (A3)
4. Host with kernel: `kernel.project.getConfig.execute().plugins`

## Specs attached

- `core:kernel`, `cli:plugins-list`
- **Possible new spec:** `core:list-plugin-declarations` (helper, not use case) — decide in design

## Sequencing

- **After** P0c (`getConfig` replaces kernel listPlugins for hosts with kernel)
- **Before** P1d audit and P1e (kernel.project slimming)
- Overlap on `core:kernel` with `01-core-kernel-get-config`, `06-core-config-editing-boundary` — serialize archives

## Rejected alternatives

- Keeping ListPlugins as thin wrapper over getConfig — rejected; still unnecessary kernel surface
