### cli:change-deps

**Source:** specIds

#### Requirements (relevant to change)

- `specd change deps <name> [<specId>] [--add <id>...] [--remove <id>...] [--set <id>...] [--format text|json|toon] [--config <path>]`
- Text output for modification: `updated deps for <specId> in change <name>` + dependsOn list
- Text output for display single: `spec dependencies for <specId> in change <name>:`
- Text output for list all: `spec dependencies for change <name>:` with per-spec bullet list

### core:update-spec-deps

**Source:** specIds

#### Requirements (relevant to change)

- `UpdateSpecDepsInput`: `name`, `specId`, `add`, `remove`, `set` (all optional except name/specId)
- Loads change by name via ChangeRepository.get; throws ChangeNotFoundError if missing
- `specId` must be in `change.specIds`; throws Error if not
- `set` mutually exclusive with `add`/`remove`; throws Error if combined
- At least one operation required; throws Error if none provided
- `set` replaces entire dependency list
- `remove` validates each ID exists; first missing ID aborts
- `add` is idempotent — duplicates silently skipped
- `remove` applied before `add` when both provided
- After update, persists via `ChangeRepository.mutate(name, fn)` using `setSpecDependsOn`
- Returns `UpdateSpecDepsResult` with `specId` and `dependsOn`
- Typed errors for validation failures (`InvalidInputError`) and not-found dependencies

#### Constraints (relevant to change)

- Updating specDependsOn does NOT trigger approval invalidation
- Does not resolve actor identity — not recorded as actor-attributed events
- Dependency spec IDs parsed via parseSpecId (lenient, does not throw on invalid)

#### Verification Scenarios (relevant to change)

| Scenario                        | Given                             | When                         | Then                                                     |
| ------------------------------- | --------------------------------- | ---------------------------- | -------------------------------------------------------- |
| execute accepts input           | —                                 | called                       | accepts UpdateSpecDepsInput                              |
| Change not found                | nonexistent                       | execute                      | throws ChangeNotFoundError                               |
| Spec ID not in change           | specIds: auth/login               | specId: billing/invoices     | throws Error                                             |
| Set + add mutually exclusive    | —                                 | both set and add             | throws Error                                             |
| Set + remove mutually exclusive | —                                 | both set and remove          | throws Error                                             |
| No operation specified          | —                                 | none of add/remove/set       | throws Error                                             |
| Set overwrites existing         | deps: [core/session]              | set: [billing, core/config]  | dependsOn = [billing, core/config]                       |
| Set with empty clears           | deps: [core/session]              | set: []                      | dependsOn = []                                           |
| Remove non-existent dep         | deps: [core/session]              | remove: [billing]            | throws Error                                             |
| Add idempotent                  | deps: [core/session]              | add: [core/session]          | dependsOn = [core/session]                               |
| Remove then add                 | deps: [core/session, core/config] | remove session + add billing | dependsOn = [core/config, billing]                       |
| Persistence                     | success                           | execute completes            | setSpecDependsOn called, ChangeRepository.mutate invoked |
| No prior state                  | no existing specDependsOn         | add: [core/session]          | dependsOn = [core/session]                               |
| Mutually exclusive typed error  | —                                 | set + add                    | InvalidInputError with INVALID_INPUT                     |
| Dep not found typed error       | deps: [core:existing]             | remove: core:non-existent    | typed error, message indicates which                     |

### core:change

**Source:** specDependsOn — defines specDependsOn field, specIds, orphan cleanup

#### Requirements (relevant to change)

- `specDependsOn` — an optional map from spec ID to an array of spec IDs representing per-spec declared dependencies. Captured during change authoring to track dependencies independently of `.specd-metadata.yaml`. Used by `CompileContext` as the highest-priority source for `dependsOn` resolution. Not subject to approval invalidation — updating `specDependsOn` does not trigger an `invalidated` event.
- `specIds` — zero or more spec IDs being created or modified by this change
- When `specIds` is updated via `updateSpecIds()`, any `specDependsOn` entry whose key is not present in the new set of spec IDs SHALL be removed (prevents orphaned entries)
