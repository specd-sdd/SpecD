# 11-sdk-host-facade — exploration

## Source

`core-refactor-on-main.md` phase **A2a**.

## Problem statement

CLI, API, IPC, MCP each duplicate `loadConfig` + `createKernel` + graph provider lifecycle. Need `@specd/sdk` thin layer: re-exports + cross-package orchestration.

## New package

`packages/sdk/` with deps on `@specd/core` and `@specd/code-graph`.

### Modules

| Export                       | Role                                                          |
| ---------------------------- | ------------------------------------------------------------- |
| `openSpecdHost`              | load config + createKernel + createSdkContext                 |
| `createSdkContext`           | kernel + createGraphProvider factory; **no duplicate config** |
| `withOpenGraphProvider`      | open/close provider lifecycle                                 |
| `buildProjectStatusSnapshot` | GetProjectSummary + GetGraphHealth                            |
| `runIndexProjectGraph`       | listWorkspaces + VCS + indexProjectGraph                      |
| re-exports                   | curated public barrels (full curation in A3)                  |

### SdkHostContext

```ts
{
  kernel: Kernel
  createGraphProvider: () => CodeGraphProvider
}
```

Config reads: `ctx.kernel.project.getConfig.execute()` (requires P0c)

Config writes (yaml-only): `createConfigWriter()` from `@specd/core` — `initProject`, `addPlugin`, `removePlugin`. **Not** kernel.project. Symmetric with `createConfigLoader()` for reads.

SDK re-exports: `createConfigLoader`, `createConfigWriter`, `createKernel` — not `createInitProject` / `createAddPlugin` / `createRemovePlugin` (removed P1e).
