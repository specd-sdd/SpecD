# 03-core-host-orchestration-context — exploration

## Source

`core-refactor-on-main.md` phase **P1a**.

## Problem statement

`change context` and `project context` CLI/API hosts build `CompileContextConfig` inline from yaml on every call. Config defaults should be baked at kernel construction; `execute` only receives runtime overrides.

## Approach

1. Internal helper `buildCompileContextConfig(config: SpecdConfig): CompileContextConfig` in `core/composition/` — **not** exported in public barrel (A3)
2. Both `CompileContext` and `GetProjectContext` store `_defaultConfig` at construction
3. `CompileContextInput` / `GetProjectContextInput` drop `config` field
4. `CompileContext`: same active-only refresh default as P0a (`refreshImplementationTracking?: boolean`)
5. Merge in execute: yaml defaults + runtime overrides

### CompileContextInput (target)

`name`, `step?`, `refreshImplementationTracking?`, `includeChangeSpecs?`, `followDeps?`, `depth?`, `sections?`, `mode?`, `fingerprint?`

### GetProjectContextInput (target)

`followDeps?`, `depth?`, `sections?`, `mode?`
