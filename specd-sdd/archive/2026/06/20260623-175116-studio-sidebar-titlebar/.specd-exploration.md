# Exploration: studio-sidebar-titlebar

## Problem statement

SpecD Studio's current shell uses a full-height `StudioTopBar` with "SpecD Studio" branding and a left sidebar that is always visible as a resizable panel with three vertically stacked sections (Changes, Workspaces-Specs, Graph). The user wants a **real collapsible sidebar** similar to Cursor/VS Code:

- Sidebar toggle button next to macOS traffic lights (or on the sidebar edge on other platforms)
- Move "SpecD Studio" branding from the top bar into the sidebar
- Decide what to show when the sidebar collapses
- Account for **Windows/Linux window controls on the right** eating top-bar buttons

## Approach / solution outline

### Two-state sidebar (not fully hidden)

| State         | Width                  | Content                              |
| ------------- | ---------------------- | ------------------------------------ |
| **Expanded**  | ~240вҖ“280px (resizable) | Full sidebar with one active section |
| **Collapsed** | ~48px fixed            | Activity rail with icons only        |

**Do NOT collapse to 0px** as the default вҖ” the sidebar is primary navigation for changes, specs, and graph. Fully hidden forces reliance on вҢҳK only.

### Activity rail (collapsed)

Three icons mapping to sidebar domains:

- **Changes** (`GitPullRequest`) вҖ” badge with active change count
- **Workspaces** (`Layers`) вҖ” optional validation-error badge
- **Graph** (`Network`) вҖ” amber dot when graph index is stale (reuse `GraphSidebarEntry` / `useGraphStatus` signal)

Behavior:

- Click icon вҶ’ expand sidebar focused on that section
- Tooltip on hover
- Active icon reflects center context (change / spec / graph open)
- **вҢҳB / Ctrl+B** toggles expanded вҶ” collapsed
- Persist width + collapsed state in `IUserStorage`

### Sidebar internal structure (expanded)

Replace the current **three vertically resizable stacked panels** with **one active section at a time**, selected via activity rail (like VS Code activity bar вҶ’ primary sidebar):

- `changes` вҶ’ existing `ChangesSidebar`
- `workspaces` вҶ’ existing `WorkspacesSidebar`
- `graph` вҶ’ existing `GraphSidebarEntry` + actions

**Sidebar header** (expanded only):

### Titlebar (lean, replaces branding area of `StudioTopBar`)

Height ~36вҖ“40px. Contents:

| Zone              | Content                                       |
| ----------------- | --------------------------------------------- |
| Left              | Sidebar toggle (+ macOS traffic-light offset) |
| Center            | Search / command palette trigger (вҢҳK)         |

**Remove** "SpecD Studio" from titlebar вҖ” moves to sidebar header.

**Nothing interactive in the rightmost ~138px on Win/Linux.**

### Overflow menu (вӢҜ) вҖ” secondary actions moved out of titlebar right cluster

To avoid collision with window controls on Win/Linux:

- Documentation (external link)
- Notifications (popover with existing system health content from `StudioTopBar`)
- Appearance (light/dark toggle)
- Future: Settings, keyboard shortcuts

On **web** (`studio-web`), all buttons MAY remain visible since there is no native window chrome.

### Toggle button placement by platform

| Platform      | Toggle position                                                                                      |
| ------------- | ---------------------------------------------------------------------------------------------------- |
| macOS         | Titlebar, immediately after traffic lights                                                           |
| Windows/Linux | Left edge of titlebar OR right edge of sidebar rail (preferred: avoids right-side conflict entirely) |
| Web           | Top-left of titlebar                                                                                 |

### Electron desktop changes

`apps/specd-studio-desktop/src/main/index.ts` вҖ” currently standard `BrowserWindow` with no `titleBarStyle`. Needs:

- `titleBarStyle: 'hiddenInset'` (macOS) or `titleBarOverlay` (Windows)
- `trafficLightPosition` on macOS
- Expose `platform` to renderer via preload for CSS safe zones

Spec: `studio-desktop:main-window-manager`

### Polling when sidebar hidden/collapsed

Existing spec `ui:hooks-workspaces-specs` verify scenario: **"Hidden sidebar skips tree poll"** вҖ” when workspace panel not visible, skip tree refetch, retain cache. Collapsed rail showing only Changes icon should skip workspace tree poll.

## Design decisions (confirmed in conversation)

1. **Collapsed = 48px icon rail**, not fully hidden
2. **Branding in sidebar header**, not titlebar
3. **One active sidebar section** instead of three vertically resizable stacked panels
4. **Safe zone right ~138px** on Win/Linux for native window controls
5. **Overflow menu (вӢҜ)** for Docs, Notifications, Theme on desktop; keeps New Change visible
6. **Toggle left-aligned** on all platforms; on Win/Linux consider sidebar-edge toggle as alternative to titlebar-right placement
7. **вҢҳB** keyboard shortcut for sidebar toggle (VS Code convention)
8. **Persist** sidebar state in `IUserStorage`

## Rejected alternatives

- **Fully hidden sidebar (0px)** вҖ” rejected because changes/specs/graph are primary nav; too much reliance on command palette
- **Keeping branding in both topbar and sidebar** вҖ” redundant
- **Keeping three vertically resizable sidebar panels** вҖ” competes with activity-rail model; cramped UX
- **Placing Docs/Bell/Theme on the right on Win/Linux without safe zone** вҖ” would be eaten by window controls (user explicitly raised this concern)
