# 04-core-host-orchestration-create — exploration

## Source

`core-refactor-on-main.md` phase **P1b**.

## Problem statement

CLI `change create` calls `getActiveSchema` before `create`. Schema resolution belongs inside `CreateChange`.

## Approach

- `CreateChangeInput`: optional `schemaName?`, `schemaVersion?`; if absent, use case calls `GetActiveSchema` internally
- Optional `includeOverlapCheck?: boolean` → `overlapReport?` in `CreateChangeResult`
- CLI drops `getActiveSchema` prelude; may pass `includeOverlapCheck: true` for stderr overlap report

## Specs attached

- `core:create-change`, `core:get-active-schema`, `cli:change-create`
