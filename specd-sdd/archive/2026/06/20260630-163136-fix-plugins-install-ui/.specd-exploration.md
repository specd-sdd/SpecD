# Exploration: fix-plugins-install-ui

## Problem statement

`specd plugins install` always routed through `InstallPlugin` and persisted under `plugins.agents`, so UI packages (`@specd/plugin-ui-studio`, `@specd/studio-web`) failed with `Use InstallUiPlugin for UI plugins`. `specd ui serve` and studio e2e could not bootstrap UI without hand-editing `specd.local.yaml`.

## Approach

Load plugin via `LoadPlugin`, branch on `plugin.type`:

- `agent` → `InstallPlugin` + `plugins.agents`
- `ui` → `InstallUiPlugin` + `plugins.ui`

Fix `toPluginBucket`: `ui` maps to `ui` (not `uis`). Already-installed detection per bucket.
