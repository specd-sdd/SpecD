# 06-core-config-editing-boundary — exploration

## Source

`core-refactor-on-main.md` phase **P1e**.

## Problem statement

`AddPlugin`, `RemovePlugin`, `InitProject` mutate `specd.yaml` via `ConfigWriter` but hang off `kernel.project` — mixing config editing with domain kernel.

## Approach

1. Add `createConfigWriter()` — mirrors `createConfigLoader()`; delivery calls port methods directly
2. Delete pass-through use cases (`InitProject`, `AddPlugin`, `RemovePlugin`) and per-op factories
3. Remove `addPlugin`, `removePlugin`, `init` from `Kernel.project`; drop `configWriter` from kernel internals
4. Retire `core:init-project` spec — requirements fold into `core:config-writer-port`
5. CLI: `createConfigWriter().initProject/addPlugin/removePlugin`; kernel only when domain ops follow
6. Add `cli:project-init` + `default:_global/architecture` to change scope
