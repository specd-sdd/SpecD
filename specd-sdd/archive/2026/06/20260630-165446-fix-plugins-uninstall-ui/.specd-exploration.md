# Exploration: fix-plugins-uninstall-ui

## Problem

`plugins install` fixed in `fix-plugins-install-ui`. `uninstall` hardcodes `removePlugin(..., 'agents', ...)`. `update` only reads `plugins.agents`. UI plugins under `plugins.ui` are not removed or updated correctly.

## Approach

- Extract `toPluginBucket` + `listDeclaredPlugins` to `plugin-bucket.ts` (shared with install).
- Uninstall: `LoadPlugin` → bucket from `plugin.type` → `removePlugin`.
- Update: enumerate `agents` + `ui` declarations.

## Specs

- `cli:plugins-uninstall`
- `cli:plugins-update`

## Note

`UninstallPlugin` / `UpdatePlugin` already handle UI in plugin-manager — CLI bucket wiring only.
