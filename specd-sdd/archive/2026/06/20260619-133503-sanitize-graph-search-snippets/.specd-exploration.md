# Problem Statement

The user reported that `specd graph search "repository" --documents` can render incorrect text-mode output when a matched snippet comes from a log file containing ANSI escape sequences. The observed output included terminal control bytes such as `\x1b[1A\x1b[K`, causing the displayed snippet block to look corrupted.

During discussion, the scope was generalized: this is not only a `--documents` problem. Any `graph search` snippet can render incorrectly if the snippet content contains ANSI escape sequences or other control characters. That means the issue can affect `symbols`, `specs`, and `documents`.

The user then expanded scope further: while touching snippet rendering, `graph search` should only print snippet blocks when the caller explicitly requests them with a new `--snippet` flag. Without `--snippet`, text output should show only the hit identity and source location metadata such as line/column where applicable.

# Agreed Scope

The change should cover all of the following:

1. Sanitize ANSI escape sequences and problematic control characters from `graph search` snippets before text-mode rendering.
2. Make snippet rendering opt-in through a new `--snippet` CLI flag.
3. When `--snippet` is omitted, do not render snippet blocks in text mode.
4. Keep location metadata visible even when snippets are omitted.
5. Update CLI documentation for the new `--snippet` flag and changed output behavior.
6. Update the `skills` package templates/instructions that mention `graph search`, so they inform agents/users that `--snippet` exists when snippet output is desired.

# Technical Interpretation

The primary defect is a CLI presentation bug, not necessarily a graph indexing bug:

- the graph may legally index textual logs as documents under current rules
- the CLI currently renders matched snippets too literally
- `normalizeSnippet` normalizes indentation but does not sanitize non-printable terminal control sequences

The expanded scope adds a product behavior change:

- snippet blocks should not be shown by default
- users/agents should opt in with `--snippet`

This likely requires updates to:

- command option parsing for `graph search`
- text rendering branches for symbols/specs/documents
- tests covering text-mode output
- CLI reference docs
- any installed skill templates or guidance that demonstrate `graph search` usage and expect snippets in default text output
