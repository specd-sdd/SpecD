# 12-cli-mcp-sdk-migration — exploration

## Source

`core-refactor-on-main.md` phase **A2b**.

## Problem statement

After SDK exists, CLI and MCP still import `@specd/core` / `@specd/code-graph` directly and duplicate orchestration. Migrate to `@specd/sdk` only.

## Approach

1. `packages/cli/package.json` and `packages/mcp/package.json`: depend on `@specd/sdk` only (remove direct core/code-graph)
2. `resolveCliContext` → thin wrapper over `openSpecdHost` (~10 lines) with CLI-specific `kernelOptions`
3. Commands delegate per reparto table:

| Command                                  | Layer                                                                                                       |
| ---------------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| change status/transition/context/approve | kernel use cases (`kernel.changes.*`; approve at `changes.approveSpec` / `changes.approveSignoff` after 09) |
| project context                          | kernel                                                                                                      |
| project init                             | `createConfigWriter().initProject()`                                                                        |
| plugins list                             | `createConfigLoader().load()` / `getConfig` only                                                            |
| plugins install/uninstall                | `createConfigWriter().addPlugin()` / `.removePlugin()`                                                      |
| project status --graph                   | buildProjectStatusSnapshot                                                                                  |
| graph stats                              | getGraphHealth + withOpenGraphProvider                                                                      |
| graph index                              | runIndexProjectGraph (+ CLI lock)                                                                           |
| graph search/hotspots/impact             | provider direct                                                                                             |

4. No regression in text/json/toon output
