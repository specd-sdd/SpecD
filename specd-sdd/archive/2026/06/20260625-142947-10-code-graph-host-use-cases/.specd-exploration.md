# 10-code-graph-host-use-cases — exploration

## Source

`core-refactor-on-main.md` phase **G1**.

## Problem statement

CLI duplicates graph orchestration (stats health, index, coverage) that should live as code-graph domain use cases.

## Use cases to implement

| Use case                | Replaces                | CLI                                                 |
| ----------------------- | ----------------------- | --------------------------------------------------- |
| `getGraphHealth`        | fetchGraphStatus        | `graph stats`                                       |
| `indexProjectGraph`     | runGraphIndex body      | used via SDK `runIndexProjectGraph`                 |
| `getSpecCoverage`       | fetchSpecCoverage       | Studio path (feature)                               |
| `getChangeSpecCoverage` | fetchChangeSpecCoverage | Studio path; uses `ChangeRepository` port from core |

## Key contracts

- `GetGraphHealthInput`: `config`, `provider`, optional `workspaces[]` for fingerprint
- `IndexProjectGraphInput`: provider, projectRoot, workspaces, graphConfig, codeGraphVersion, vcsRef?, force?, onProgress?
- `getChangeSpecCoverage(provider, changes, { changeName })` → `ChangeNotFoundError` from core if missing

## Direct provider (no use case)

`graph search`, `hotspots`, `impact` — provider methods only, re-exported via SDK
