# 02-core-host-orchestration-status — exploration

## Source

`core-refactor-on-main.md` phases **P0a** + **P0b**.

## Problem statement

CLI `change status` and `change transition` manually do:

```ts
const active = await kernel.changes.repo.get(name)
if (active !== null) {
  await kernel.changes.refreshImplementationTracking.execute({ name })
}
```

This orchestration belongs in domain use cases with explicit defaults.

## Approach

### P0a — GetStatus

Extend `GetStatusInput`:

- `refreshImplementationTracking?: boolean` — default `true` when omitted
- When true: refresh **only if change is active** (`repo.get(name) !== null`)
- Draft/discarded/archived: never refresh
- `ifModifiedSince` evaluated after refresh when refresh runs

CLI `packages/cli/src/commands/change/status.ts`: remove manual refresh block. Note: CLI `--implementation` flag is display-only, does not control refresh.

### P0b — TransitionChange

Add `refreshImplementationTrackingBefore?: boolean` (default true, active-only).

CLI `packages/cli/src/commands/change/transition.ts`: remove manual refresh prelude.
