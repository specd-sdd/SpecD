# 09-core-approval-gates-baked — exploration

## Source

`core-refactor-on-main.md` phase **P3**.

## Problem statement

1. `TransitionChange`, `ApproveSpec`, `ApproveSignoff` require hosts to re-pass `approvalsSpec` / `approvalsSignoff` on every `execute` though kernel was built with `config.approvals`. `GetStatus` already bakes approvals correctly.
2. `ApproveSpec` / `ApproveSignoff` are change-lifecycle gates but exposed as `kernel.specs.approveSpec` / `kernel.specs.approveSignoff` — wrong namespace vs CLI (`specd change approve`) and domain (`ChangeRepository`).

## Approach

Bake at construction (same shape as GetStatus):

```ts
type ApprovalGates = { readonly spec: boolean; readonly signoff: boolean }
```

Relocate kernel entries:

```ts
kernel.changes.approveSpec // was kernel.specs.approveSpec
kernel.changes.approveSignoff // was kernel.specs.approveSignoff
```

- `TransitionChangeInput`: remove `approvalsSpec`, `approvalsSignoff`; keep P0b `refreshImplementationTrackingBefore?`
- `ApproveSpecInput` / `ApproveSignoffInput`: only `name`, `reason`
- `createKernel` / standalone factories pass `config.approvals`
