# Exploration: local-vendored-sqlite-runtime

## Problem statement

`packages/code-graph-electron/vendor/better-sqlite3/` is currently committed to git
(including JS sources copied from npm and a prebuilt `better_sqlite3.node` for macOS
arm64). This creates redundancy (sources are already available via `better-sqlite3` in
`node_modules`), pollutes `git status` with node-gyp build intermediates after local
rebuilds, and commits a platform-specific native binary that does not work on Linux or
Windows.

The user wants the **cleanest solution without committing vendor to git**, while
preserving the existing Electron native-runtime isolation that motivated vendoring in
the first place.

## Approach / solution outline

Keep the **vendored physical copy pattern** (required for pnpm path isolation — aliases
collapse to the same `.pnpm/better-sqlite3@...` store path). Stop tracking `vendor/` in
git; generate it locally via existing scripts.

Target architecture:

```
git:     code + scripts only
disk:    vendor/ (generated on demand)
runtime: Electron loads vendor/.../better_sqlite3.node (compiled locally per platform)
```

### Implementation phases (agreed in conversation)

1. Add `packages/code-graph-electron/vendor/` to `.gitignore` and remove tracked vendor
   files from git index (without deleting local copies).
2. Keep `sync:vendor-sqlite` on `code-graph-electron` build — copies canonical
   `better-sqlite3` from `node_modules` into `vendor/`.
3. Keep conditional rebuild on `studio-desktop` `prestart` and `build` (already wired via
   `rebuild:graph-sqlite-electron`).
4. Improve rebuild cache metadata in `.electron-build.json`: store
   `electronVersion`, `platform`, and `arch` instead of absolute `binaryPath` (current
   metadata contains machine-specific absolute paths).
5. Do **not** add a monorepo-wide `postinstall` rebuild — CLI/API contributors should
   not pay native compile cost.
6. Ensure CI jobs that build `studio-desktop` run the rebuild step explicitly.
7. Optionally gitignore node-gyp intermediates under `vendor/.../build/**` except the
   final `.node` if any vendor files remain partially tracked during transition.

## Design decisions

### Confirmed / agreed

- Vendor directory stays as a **local generated artifact**, not a git-tracked tree.
- Physical copy isolation from pnpm store is **non-negotiable** — do not revert to npm
  alias or shared `node_modules` resolution for Electron.
- Rebuild should remain **lazy/conditional** and scoped to desktop workflows.
- Native binary must be compiled **per platform** locally or in CI — not committed.

### Rejected alternatives

| Alternative                            | Reason rejected                                       |
| -------------------------------------- | ----------------------------------------------------- |
| Commit only `.node`                    | Platform-specific; wrong binary on other OS/arch      |
| Commit entire `vendor/` (status quo)   | Redundant with npm sync; noisy git status             |
| Remove vendor; use npm install only    | pnpm path collapse; Electron/Node share addon         |
| `pnpm` alias `better-sqlite3-electron` | Proven to resolve to same `.pnpm` store path          |
| Global `postinstall` rebuild           | Slow; requires native toolchain for all monorepo devs |
