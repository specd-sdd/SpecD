# Exploration Context: fix-project-dashboard-snapshot

Generated: 2026-07-20

## Problem Statement

The `specd project dashboard` command does not report the same metrics as `specd project status`.
Specifically:

- `dashboard` queries `kernel.specs.list`, `kernel.changes.list`, `kernel.changes.listDrafts`, and `kernel.changes.listDiscarded` manually instead of using `buildProjectStatusSnapshot` / `getProjectSummary`.
- This leads to discrepancies in total spec counts (e.g. 241 in dashboard vs 243 in status) and missing empty workspaces in spec breakdowns.
- `dashboard` missing counts for archived changes.
- `dashboard` lacks graph health diagnostic information (freshness, file/symbol counts) available in `buildProjectStatusSnapshot`.
- The TUI layout in `dashboard.ts` has wrapping bugs: when a project has many workspaces (e.g., 15 in specd), the `workspaces:` row overflows the right border of the inner/outer boxes instead of wrapping gracefully across multiple indented lines.

## Approach / Solution Outline

1. Update `registerProjectDashboard` in `packages/cli/src/commands/project/dashboard.ts` to fetch data via `buildProjectStatusSnapshot(host, { includeGraph: true })`.
2. Support full JSON/TOON output schema matching the new capabilities (specs total/byWorkspace, changes active/drafts/discarded/archived, and optional graph metadata).
3. Update wrapping logic in `dashboard.ts` so that both `root:` and `workspaces:` wrap properly across multiple lines without overflowing box borders.
4. Expand the TUI layout to include an inner box for `Graph` health (freshness, file/symbol count) or update box layout to be responsive and full-width where appropriate.
5. Update `specs/cli/project-dashboard/spec.md` and `verify.md` to document the new requirements, data sources, schema fields, and layout wrapping behavior.

## Design Decisions

- `dashboard` will use `buildProjectStatusSnapshot` as the canonical source of truth for project status, aligning with `specd project status`.
- In `text` mode, `workspaces:` line will wrap across multiple lines indented to align with the value column when it exceeds available box width.
- In `json`/`toon` mode, output will include `changes.archived` and `graph` health information when available.

## Key Codebase Observations

- `packages/cli/src/commands/project/status.ts` uses `openSpecdHost` and `buildProjectStatusSnapshot`.
- `packages/sdk/src/orchestration/build-project-status-snapshot.ts` provides `summary`, `graphHealth`, `approvals`, `llmOptimizedContext`.
- `packages/cli/src/commands/project/dashboard.ts` currently computes counts using `kernel.specs.list.execute()`.
