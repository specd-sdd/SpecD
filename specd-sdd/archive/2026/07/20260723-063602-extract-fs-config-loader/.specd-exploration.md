# Exploration Context: extract-fs-config-loader

Generated: 2026-07-23

## Problem Statement

The adapter file `packages/core/src/infrastructure/fs/config-loader.ts` currently spans over 1,500 lines of code. It conflates three major responsibilities:

1. **Domain-level configuration structures**: Final validation Zod schemas (`SpecdYamlZodSchema`, `WorkspaceRawZodSchema`, etc.) and pattern syntax validation (`validateContextPattern`).
2. **FS-specific cascade logic**: Reading, parsing, resolving, merging, and applying removals to layered config variants (e.g. `specd.local.yaml`, circular detection, cascade discovery).
3. **FS Adapter plumbing**: The `FsConfigLoader` class which implements the `ConfigLoader` port.

## Refactoring Strategy

### 1. Domain Configuration Schema Isolation

Move domain-level schemas and validators from `packages/core/src/infrastructure/fs/config-loader.ts` to `packages/core/src/application/ports/config-schema.ts` (or equivalent location in `application/ports` or `domain/`).

- Move `SpecdYamlZodSchema` and all sub-schemas (`WorkspaceRawZodSchema`, `ProjectGraphZodSchema`, etc.).
- Move `validateContextPattern` and `validateContextPatterns`.
- Move `deepMergeRawConfig` (or check if it should remain a helper).

### 2. Cascade Logic Extraction

Extract the filesystem-specific layered cascade logic into a dedicated module inside the `fs` infrastructure folder, e.g. `packages/core/src/infrastructure/fs/config-cascade.ts`.

- Move `LayerRawZodSchema`, `LayerExtendsZodSchema`, `LayerRemovalZodSchema`.
- Move discovery functions: `candidateSortKey`, `discoverCandidateFiles`, `findCandidateDirectory`.
- Move parsing and chain resolution: `parseCascadeLayer`, `resolveActiveChain`, `resolveForcedCascade`, `mergeActiveLayers`.
- Move removal logic: `applyRemovals`, `matchesContextEntry`, `findContextEntryIndex`.
- Move environment loading: `tryLoadEnvFiles`, `applyEnvOverrides`.

### 3. FsConfigLoader Clean-Up

Keep only the `FsConfigLoader` class definition in the adapter file.

- It will import the cascade resolution helper from `config-cascade.ts`.
- It will validate the merged config using the Zod schemas from the application layer.
