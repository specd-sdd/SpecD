# Exploration Context: normalize-repository-caches

Generated: 2026-07-16

## Problem Statement

Currently, `FsArchiveRepository` maintains its cache (`.specd-index.jsonl` and `.specd-index-meta.json`) inside the archive directory itself. Other repositories (`FsChangeRepository` and `FsSpecRepository`) do not have caching, which leads to expensive directory walks and manifest parsing on listing operations (`list()`, `count()`).

We need to normalize the cache strategy across these repositories, move the cache files to a centralized folder under `{configPath}/tmp/fs-cache/`, and design a highly efficient, stream-based incremental invalidation strategy.

## Key Design Decisions & Agreements

### 1. Cache Locations

All cache databases (`.specd-index.jsonl` and `.specd-index-meta.json`) will live under:

- `{configPath}/tmp/fs-cache/archive/`
- `{configPath}/tmp/fs-cache/changes/`
- `{configPath}/tmp/fs-cache/drafts/`
- `{configPath}/tmp/fs-cache/discarded/`
- `{configPath}/tmp/fs-cache/specs/[workspace]/`

### 2. Selective Invalidation

We do NOT invalidate/rebuild on every write. We only trigger invalidation/updates for:

- **Changes/Drafts/Discarded**: Creation, state transition, description update, or specIds scope changes. (We skip invalidation on artifact saves/skips/validations).
- **Specs**: Spec creation, publishing, deletion, or lock/metadata updates.

### 3. Freshness Verification & Scaling

To handle git pulls/checkouts or external modifications without loading the entire database into memory:

- **Hybrid Check**: Use a TTL (e.g. 5 minutes) and the VCS commit hash (`VcsAdapter.ref()`) stored in `.specd-index-meta.json` as a safeguard.
- **Stream-Based Incremental Rebuild**:
  1. Scan disk to build a lightweight `Set<string> diskEntries`.
  2. Stream the original `.specd-index.jsonl` line-by-line.
  3. Compare file `mtimes` on disk:
     - _Changes/Archive_: check `mtime` of `manifest.json`.
     - _Specs_: since there is no manifest, the cache stores `files: Array<{filename, mtime, hash}>`. We `fs.stat()` each file inside the spec directory.
     - **Cache Hit**: Write the original line directly to a `.tmp` file (O(1) memory, O(1) parsing).
     - **Cache Miss**: Read and parse the updated file, write the new line.
  4. Write new/unprocessed entries at the end.
  5. Atomically rename the `.tmp` file.
- **Vcs-Based Differential Fast-Path**: For large repositories (like `archive` with thousands of entries), if commit changes, use `vcsAdapter.modifiedFiles(cachedRef)` to identify the modified files first. If no files in the archive path changed, simply update the cached commit ref. Fall back to incremental `fs.stat` check for external/untracked paths.
