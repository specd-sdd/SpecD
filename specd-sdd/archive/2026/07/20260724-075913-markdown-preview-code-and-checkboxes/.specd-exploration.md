# Exploration: markdown-preview-code-and-checkboxes

Generated: 2026-07-24

## Problem statement

In Studio UI markdown preview, three visual gaps:

1. **Code blocks lack syntax highlighting** ‚Äî fenced/inline code renders in the same foreground color as body text (white in dark mode, black in light mode). Users want distinct token colors.
2. **Checked GFM task-list checkboxes look gray** ‚Äî when checked they appear gray, not green. Simply coloring the native checkbox fails because browsers force `input[type=checkbox]:disabled` (or equivalent disabled task inputs from remark-gfm) into a grayed appearance. Need a non-native (or non-disabled-native) presentation for checked state.
3. **Mermaid fenced blocks are not diagrams** ‚Äî ` ```mermaid ` ``shows as plain code; users want rendered diagrams in the shared preview (lazy-loaded, theme-aware, with failure fallback). Change name stays`markdown-preview-code-and-checkboxes`.

## Approach / solution outline

- Shared Studio markdown surface (`.studio-markdown-preview` + `react-markdown` + `remark-gfm`) should gain:
  - **Syntax highlighting** for fenced code via `rehype-highlight` + highlight.js; GitHub Dark/Light by Studio theme.
  - **Task checkboxes**: Lucide icons + `text-studio-success` when checked; not styling disabled native inputs.
  - **Mermaid**: fenced `mermaid` blocks rendered with lazy-loaded `mermaid`, dark/light theme, source+error fallback on failure.
  - Prefer one shared renderer for inspector, Spec context, and Tasks.
